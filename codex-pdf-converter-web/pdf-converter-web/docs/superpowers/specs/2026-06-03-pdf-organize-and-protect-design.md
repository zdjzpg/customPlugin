# PDF 页面整理与密码工具设计

## 目标

新增 3 个主流高频 PDF 工具：

- 删除 PDF 页面
- 调整 PDF 页面顺序
- 保护 PDF / 解锁 PDF

要求：

- 符合主流市场常见能力边界
- 继续沿用当前买家页工具箱模式
- 功能可卖、可测、可上线

## 已确认范围

### 工具拆分方式

采用：

- 3 个独立工具

分别是：

1. `删除 PDF 页面`
2. `调整 PDF 页面顺序`
3. `保护 PDF / 解锁 PDF`

不做：

- 合并成一个“万能编辑器”
- 页面级编辑和密码工具混在同一个表单里

## 删除 PDF 页面

### 第一版能力

- 输入一个 PDF
- 支持两种操作方式：
  - 页码输入
  - 缩略图页级选择
- 选择要删除的页面
- 输出一个新的 PDF

### 页码输入

支持：

- `1,3,5-8`

### 缩略图模式

- 展示每页缩略图
- 用户可点击选择要删除的页

## 调整 PDF 页面顺序

### 第一版能力

- 输入一个 PDF
- 支持两种方式：
  - 页码输入
  - 缩略图拖拽
- 输出一个新的重排 PDF

### 页码输入

- 允许用户输入新的页序
- 例如：
  - `3,1,2,4`

### 缩略图拖拽

- 展示每页缩略图
- 用户可拖拽排序
- 结果按最终排序导出

## 保护 PDF / 解锁 PDF

### 第一版能力

只做：

- `打开密码`

支持：

- `保护 PDF`
  - 设置密码
  - 导出需输入密码才能打开的新 PDF
- `解锁 PDF`
  - 输入已有密码
  - 导出无密码 PDF

第一版不做：

- 权限密码
- 禁止打印
- 禁止复制
- 更细粒度安全策略

## 交互设计

### 删除 PDF 页面 / 调整 PDF 页面顺序

桌面端：

- 详情页上方提供：
  - 页码输入模式
  - 缩略图模式
- 用户可切换模式

移动端：

- 仍然用移动端独立 UI 详情页
- 页码输入为主
- 缩略图模式如果可行则保留；若拖拽体验不稳，优先保证能点击重排或切换顺序

### 保护 PDF / 解锁 PDF

详情页提供两个子模式：

- 保护 PDF
- 解锁 PDF

保护模式字段：

- 文件上传
- 密码输入
- 确认密码

解锁模式字段：

- 文件上传
- 原密码输入

## 技术路线

### 删除页面 / 调整顺序

推荐继续使用：

- `pypdf`

理由：

- 已在当前项目中用于页面级操作
- 和现有 `提取页面 / 拆分 / 合并 / 旋转` 思路一致

缩略图展示可继续复用：

- `pdf2image + poppler`

### 保护 / 解锁

推荐使用：

- `pypdf`

理由：

- 可处理 PDF 加密/解密
- 与当前项目依赖一致

## 范围外

这一轮不做：

- 多 PDF 批量删除/排序
- 页级多选框和拖拽的可视化高级编辑器
- 权限密码
- 更复杂的 PDF 安全设置
- 云端异步任务队列

## 测试要求

至少覆盖：

- catalog 暴露 3 个新工具
- 删除页面服务层真实输出
- 页序调整服务层真实输出
- 加密/解锁服务层真实输出
- 路由参数透传
- 前台详情页渲染
- 浏览器真实流程至少覆盖：
  - 删除页面
  - 调整顺序
  - 保护 PDF
  - 解锁 PDF
