# PDF 转 PPTX 设计

## 目标

新增一个纯开源 `PDF 转 PPTX` 工具：

- 输出 `.pptx`
- 以 `1 页 PDF = 1 张 PPT 幻灯片` 为固定规则
- 优先把文字和图片拆成可编辑内容
- 不承诺复杂版式高保真

## 已确认范围

### 转换方向

- 输入：单个 `.pdf`
- 输出：单个 `.pptx`

### 第一版目标

- 文本型 PDF：
  - 尽量提取文字块
  - 生成可编辑文本框
- 扫描件 PDF：
  - 自动判断文本不足时先做 OCR
  - 再生成可编辑文本框
- 页面内图片：
  - 尽量独立提取
  - 放进对应幻灯片

### 第一版不承诺

- 表格结构还原
- 多栏复杂排版精确对齐
- 矢量图形还原
- 批注、艺术字、页眉页脚精确重建
- 页级智能混合分流

## 用户交互

买家页新增一个工具卡：

- `PDF 转 PPT`

详情页第一版保持简单：

- 选择一个 PDF
- 点击开始转换
- 不暴露模式切换

文案需要明确：

- 适合把常见 PDF 内容快速整理成可修改 PPT
- 复杂排版可能会有偏差

## 技术路线

采用纯开源方案：

- `PyMuPDF`
  - 提取页面尺寸
  - 提取文字块、图片块和坐标
- `python-pptx`
  - 创建 PPTX
  - 按页生成幻灯片
  - 写入文本框和图片
- `ocrmypdf`
  - 当原 PDF 可提取文本明显不足时，先生成可搜索 PDF

## 自动判断规则

第一版按整份文档做判断：

- 先读取各页文字抽取结果
- 如果整份文档几乎抽不到文本，则视为扫描件
- 扫描件先 OCR，再进入 PPT 生成

不做：

- 同一份 PDF 内逐页切换 OCR / 非 OCR 管线

## PPT 生成规则

- 使用第一页尺寸作为整份 PPT 的幻灯片尺寸基准
- 每页 PDF 映射为一张幻灯片
- 文本块映射为文本框
- 图片块映射为图片
- 坐标按页面比例换算到幻灯片坐标

### 失败回退

为了避免用户空跑：

- 若某页内容抽取异常，可退化为整页截图放入该幻灯片
- 若整份文件仍能生成 PPTX，则按成功返回

## 测试要求

至少覆盖：

- catalog 暴露 `pdf_to_pptx`
- 路由能转发上传请求
- 服务层能生成 `.pptx`
- 文本型 PDF 生成的 PPT 含有文本内容
- 扫描件路径能走 OCR 预处理并输出 `.pptx`
- 前台详情页渲染正确

服务器需要新增或确认：
