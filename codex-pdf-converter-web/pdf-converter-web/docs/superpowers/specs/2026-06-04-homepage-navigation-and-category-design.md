# 首页导航与分类页设计

## 目标

在保持买家卡密登录门槛不变的前提下，为当前工具站补一个登录后的首页导航层。

这一轮只做：

- 登录后分类首页
- 单个分类入口
- 分类下工具列表页
- 工具卡 hover 上浮特效
- 移动端可用

这一轮不做：

- 多分类完整体系
- 文本工具实现
- 搜索 / 热门 / 收藏 / 历史
- 直接复用外站图标素材

## 已确认范围

### 登录规则

- 未登录时，仍然只展示当前卡密登录页
- 只有买家登录成功后，才允许浏览工具导航和分类页

### 分类范围

当前只展示一个已实现分类：

- `PPT 工具`

注意：

- 这个分类名只是当前统一入口名称
- 该分类页下会暂时承载当前所有已实现工具
- 这是一个过渡性结构，后续可以再拆分成 `PDF 工具`、`图片工具`、`文本工具` 等

### 页面层级

登录后买家端改为三级结构：

1. `分类首页`
   - 只展示一个分类卡：`PPT 工具`
2. `分类工具页`
   - 展示当前所有已实现工具卡
3. `工具详情页`
   - 继续复用当前上传、参数、转换、下载流程

## 交互设计

### 分类首页

- 显示品牌标题与简短说明
- 显示一个大分类卡 `PPT 工具`
- 点击分类卡后进入分类工具页

### 分类工具页

- 显示分类标题
- 显示 `返回分类首页`
- 工具卡沿用当前概览 -> 详情逻辑
- 点击工具卡上的 `查看详情` 后进入现有工具详情页

### 工具详情页

- 保持当前行为不变
- 继续显示 `返回列表`
- 返回时回到当前分类工具页，而不是直接回到登录页

## 视觉设计

### 整体方向

- 不直接照搬 UU 在线工具的素材和布局
- 参考其“工具站导航感”，但保留当前项目已有的暖色玻璃卡片风格

### 分类卡与工具卡特效

hover 反馈统一采用：

- 轻微上浮
- 阴影增强
- 边框高亮
- 柔和背景光晕

不做：

- 夸张 3D 翻转
- 大幅位移
- 花哨粒子效果

### 图标策略

- 不直接抓取或复制外站图标资源
- 使用自有简单图形或开源图标风格替代
- 当前只保证分类卡有清晰图标
- 工具卡图标不是这一轮必做项

## 技术方案

### 数据模型

前端新增分类配置层。

当前建议结构：

- `category key`
- `category label`
- `category description`
- `category icon token`
- `tool keys`

后端 catalog 仍然只负责返回真实可用工具列表。

前端根据：

- 后端 catalog
- 本地分类配置

组合出分类首页和分类工具页。

### 状态模型

现有视图状态从：

- `overview`
- `detail`

扩展为至少：

- `categories`
- `category_tools`
- `detail`

并增加：

- `categoryKey`
- `conversionKey`

用于在分类页与详情页之间回退。

### 渲染职责

建议拆分为：

- 分类首页 markup
- 分类工具页 markup
- 现有工具详情 markup

现有 `toolCatalogMarkup.mjs` 可以继续承担工具卡和详情页渲染，但分类卡建议单独加一个轻量渲染模块或同文件内清晰拆段。

### 移动端策略

移动端继续保留当前独立 UI 思路：

- 分类首页显示单列大卡
- 分类工具页显示单列工具卡
- 详情页继续走当前全屏感流程

不单独为这一轮引入新的移动端信息架构。

## 测试要求

至少覆盖：

- 登录后默认进入分类首页
- 分类首页显示 `PPT 工具`
- 点击分类后展示当前所有已实现工具
- 点击工具后进入现有详情页
- `返回分类首页` / `返回列表` 路径正确
- 工具详情现有流程不回归
- 移动端分类页渲染正确

## 后续待办

- 首页搜索
- 工具图标体系扩展
- 热门 / 最新 / 收藏 / 历史
