# UU 风格买家端 UI 重构设计

## 目标

将当前买家端从“单页工具面板”重构为更接近 UU 在线工具的信息架构和视觉系统。

重构目标：

- 桌面端向 UU 桌面结构靠拢
- 移动端向 UU 移动端结构靠拢
- 登录页也纳入同一套工具站风格
- 保留现有卡密登录、工具详情、上传转换、结果下载业务流程

## 本轮范围

### 包含

- 买家登录页
- 买家登录后首页
- 桌面端顶部栏
- 桌面端左侧分类导航
- 桌面端主内容区
- 移动端顶部栏
- 移动端分类入口
- 搜索区与快捷关键词区
- 工具列表卡片
- 工具详情页壳层

### 不包含

- 管理后台 UI
- 后端接口调整
- 转换逻辑调整
- 新工具实现
- 历史 / 收藏 / 热门 / 最新的真实数据功能

## 已确认约束

### 登录门槛

- 未登录用户仍然不能浏览工具
- 必须先输入卡密并登录
- 登录成功后进入新的买家工具站首页

### 当前分类策略

这轮虽然整体骨架按多分类工具站来设计，但只展示一个真实分类：

- `PPT 工具`

当前所有已实现工具暂时都放在这个分类下。

这是阶段性组织方式，不代表最终分类模型。

### 平台范围

- 桌面端与移动端都要一起重做
- 两端分别向 UU 的对应形态靠拢
- 不是简单响应式缩放同一套布局

## 信息架构

### 桌面端

桌面端采用三层骨架：

1. `顶部固定栏`
   - 左侧品牌
   - 右侧轻导航按钮
2. `左侧分类导航`
   - 当前只显示一个真实分类：`PPT 工具`
3. `主内容区`
   - 搜索框
   - 快捷关键词
   - 当前分类标题
   - 工具卡栅格
   - 工具详情容器

登录后默认进入：

- `PPT 工具` 的工具列表页

不再停留在当前那种只有一个大分类卡的过渡页。

### 移动端

移动端采用独立结构：

1. `顶部栏`
   - 左侧品牌
   - 右侧菜单按钮
2. `搜索区`
   - 搜索输入
   - 快捷关键词
3. `当前分类区`
   - 当前分类标题
   - 单列工具卡流
4. `详情页`
   - 保留当前全屏感，但外壳与节奏改成 UU 移动端风格

移动端分类入口通过：

- 顶部菜单
- 或轻量抽屉 / 折叠层

来承载，而不是桌面左栏直接缩放。

## 页面行为

### 登录页

- 仍然以卡密登录为唯一入口
- 但视觉上改成工具站式页面
- 去掉当前营销型大 Hero 感
- 保留品牌说明和简单提示

### 登录后首页

- 桌面端显示完整工具站骨架
- 移动端显示移动版工具站首页
- 默认定位在 `PPT 工具`

### 分类切换

这轮只显示一个真实分类，因此不会出现多分类切换行为复杂度。

但前端状态模型必须允许未来扩展到多分类。

### 搜索

这轮搜索必须是可用的，不做纯空壳。

第一版规则：

- 仅前端本地筛选当前已实现工具
- 匹配字段：
  - 工具名
  - helperText

### 工具详情页

- 继续复用现有工具详情和转换逻辑
- 只重做壳层、标题、回退、布局、间距、导航关系
- 返回逻辑保持：
  - 返回当前工具列表
  - 不回到登录页

## 视觉设计

### 总体方向

目标是“几乎换皮”到 UU 风格，但不直接复制其素材资源。

需要对齐的层面：

- 版式结构
- 字号层级
- 导航节奏
- 卡片密度
- 页面留白
- 阴影强度
- 边框与圆角
- hover / active 反馈

### 需要抛弃的现状

当前买家端存在这些不再保留的风格：

- 大型 Hero 主标题
- 偏营销页的首屏氛围
- 暖色玻璃感背景
- 过大的字号和标题比例
- 过于稀疏的工具卡密度

### 新视觉基调

- 更干净的浅灰白背景
- 更接近工具站的密度和秩序
- 字体缩小一档
- 标题、正文、提示、标签的层级更像 UU
- 卡片采用轻阴影与细边框
- 导航和内容区边界更明确

### 图标策略

- 不直接抓取或复制 UU 图标资源
- 使用开源图标或自绘简单图形替代
- 风格上与 UU 接近
- 本轮至少覆盖：
  - 顶栏品牌图形
  - 分类导航图标
  - 工具卡的小图标位

## 状态模型

买家端状态至少需要支持：

- `login`
- `tool_list`
- `detail`

并保留：

- `categoryKey`
- `conversionKey`
- `searchKeyword`

桌面与移动端共享核心状态，但渲染层允许分流。

## 前端结构建议

### 页面布局

建议将买家端页面拆成以下职责单元：

- 顶栏 markup
- 桌面侧栏 markup
- 移动端导航 markup
- 搜索区 markup
- 分类工具列表 markup
- 工具详情容器 markup

### 数据组织

后端 catalog 仍返回真实工具列表。

前端新增：

- 分类配置
- 顶栏按钮配置
- 搜索热词配置

当前虽然只有一个真实分类，但结构上仍按“可扩展多分类”处理。

## 测试要求

至少覆盖：

- 登录成功后进入新的买家工具站首页
- 桌面端显示顶部栏、左侧分类区、主内容区
- 移动端显示顶部栏、菜单/分类入口、搜索区、工具流
- 搜索能对当前工具进行前端筛选
- 点击工具后仍能进入现有详情页
- 详情页返回后回到工具列表
- 现有上传、转换、下载交互不回归

## 已确认后续待办

这一轮不实现，但需要保留扩展空间：

- 文本工具线
- 真实多分类拆分
- 热门 / 最新 / 历史 / 收藏真实功能
- 更完整的图标体系
